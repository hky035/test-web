# sos-web

## 📌 Convention

### branch naming
```
main - develop - feature/*
```

### commit message 
|type|description|
|:-:|---|
|feat|새로운 기능 개발, 요구사항을 반영한 기능 수정|
|fix|기능에 관한 버그 수정|
|style|코드에 변화가 없는 코드 스타일, 포맷팅에 관한 수정|
|refact|기능 변화가 아닌 코드 리팩토링|
|test|테스트 코드 추가/수정|
|chore|그 외 수정사항 ex) 패키지매니저, 파일/디렉토리 이름・위치 변경|
|docs|문서・주석 수정|
